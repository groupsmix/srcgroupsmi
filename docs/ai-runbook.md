# AI Systems Runbook

This runbook covers operational procedures for our AI pipelines (Chat, Jobs, Store, Articles) and how to respond to incidents when external providers fail.

## 1. Provider Outages (OpenRouter, Groq, Anthropic, OpenAI)

The AI architecture is highly resilient, but total provider failure or quota exhaustion requires manual intervention.

**Detection:**
- **Sentry Alerts:** High frequency of `500 Internal Server Error` from `/api/chat` or `functions/api/groq.ts`.
- **Axiom Logs:** Circuit Breaker logs showing `jobs-board: OpenRouter circuit breaker open`.
- **User Reports:** Responses hanging or failing completely.

**Resolution Steps:**
1. **Check Provider Status:**
   - Groq: https://status.groq.com
   - OpenRouter: https://openrouter.ai/activity
   - Anthropic: https://status.anthropic.com
   - OpenAI: https://status.openai.com
2. **Force Paid Fallback:** If free tiers (Groq/OpenRouter) are exhausted or down, ensure the `ANTHROPIC_API_KEY` or `OPENAI_API_KEY` is set in Cloudflare Worker environment variables. The fallback in `groq.ts` will trigger automatically.
3. **Disable Circuit Breakers (Emergency Only):** If a provider is flapping and the circuit breaker is falsely tripping, remove the `checkCircuitBreaker` logic in `chat.js` or `jobs-board.js` and deploy a hotfix, but **only if** you can guarantee we won't exceed paid API limits.

## 2. Quota Management

User AI usage is capped daily by the `checkAndConsumeQuota` function (`_shared/ai-quota.js`).

**Detection:**
- **Axiom Logs:** Spike in `429 Too Many Requests` for AI endpoints (`/api/chat`, etc.).
- **Supabase:** The `ai_audit` table shows a massive influx of queries from a single `user_id`.

**Resolution Steps:**
1. **Investigate Abuse:** Query the `ai_audit` table for the top users by token usage over the last 24h.
2. **Increase Daily Quota:** To globally raise the free-tier quota (e.g., during a launch or promotion), update the `AI_QUOTA_DAILY_LIMIT` environment variable in Cloudflare Pages.
3. **Ban Abusive Users:** If a user is attempting to bypass limits or scrape the AI endpoints, ban them via Supabase Auth.

## 3. Moderation Evasion & Jailbreaks

We use a combination of `Llama Guard` (via `moderateOutput`) and explicit system prompt wrappers (`withUserInputDirective`) to prevent jailbreaks.

**Detection:**
- **Axiom Logs:** Search for `Llama Guard returned 500` (fails open) or successful blocks.
- **User Reports:** Inappropriate or rule-breaking content generated by the AI.

**Resolution Steps:**
1. **Review Logs:** Pull the user's prompt from the `ai_audit` table.
2. **Tune System Prompt:** Update the `SYSTEM_PROMPT` in `chat.js` or `article-ai.js` to explicitly instruct the model to refuse the specific jailbreak technique.
3. **Enforce Hard Bans:** If a user is repeatedly attempting jailbreaks, suspend their account.
